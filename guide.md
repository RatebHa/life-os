# Life OS Guide

Life OS works best as a commitment system, not as a storage bin.

Use it to:
- decide what matters
- keep today realistic
- protect focus
- stay honest with habits
- review what is slipping
- recover without losing trust in the system

If you approach the app this way, it becomes much more useful:

1. Capture quickly.
2. Clarify what the item really is.
3. Commit only to what should be active.
4. Keep `Today` small and believable.
5. Review what keeps slipping.
6. Recover by reducing scope, not by pretending.

## 1. How To Think About Life OS

Life OS is built around a simple stack:

- `Domains` = the parts of your life you care about
- `Goals` = what direction matters inside those domains
- `Tasks` = specific commitments
- `Habits` = repeated behaviors
- `Notes` = thinking, planning, and reflection
- `Today` = execution
- `Overview` = orientation
- `Weekly Review` = recalibration

Good rule:
- Goals give direction.
- Tasks create movement.
- Habits create consistency.
- Today decides what is real now.

Do not try to put your whole life into the app on day one.
Build a system you can trust first.

## 2. The Right Way To Approach The App

Approach Life OS in layers:

### Layer 1: Build the structure
- Create your domains.
- Add only your real active goals.
- Add the habits you actually want to keep.

### Layer 2: Build your current commitments
- Capture tasks that truly matter.
- Give tasks a domain.
- Use due dates sparingly.
- Use start dates to stage work instead of activating everything now.

### Layer 3: Run your day from `Today`
- Pick one MIT.
- Keep the Top 3 short.
- Start focus quickly.

### Layer 4: Learn from reality
- Log friction honestly.
- Use habit skip reasons honestly.
- Run Weekly Review.
- Adjust the system based on what actually happened.

That is the real loop.

## 3. First Setup

Recommended first setup:

1. Create `3-5 domains`.
2. Open `Settings`.
3. Set your backup folder.
4. Turn on auto backup if you want daily protection.
5. Create one manual backup immediately.
6. Tune `CRT intensity`, `text scale`, and `UI density`.
7. Add only a few active goals, tasks, and habits.

Best practice:
- start small
- do not import a giant backlog
- only add what you are willing to review and maintain

If you use multiple machines:
- store backups in a synced folder like Syncthing
- sync backup files, not the live SQLite database

## 4. Your First 10 Minutes In The App

If you are new to the app, do this:

1. Create your domains.
2. Add `1-3` real goals that matter now.
3. Add a few current tasks.
4. Add `1-3` habits you actually want to keep.
5. Open `Today`.
6. Choose one MIT.
7. Set the Top 3.
8. Start one focus block.

That is enough to make the app useful immediately.

## 5. The 30-Second Daily Start

Most days should begin like this:

1. Open `Today`.
2. Confirm your MIT.
3. Confirm your Top 3.
4. Check whether the board still feels realistic.
5. Start the first focus block.

That is the core value of Life OS.

Do not spend the first 20 minutes reorganizing everything.
Get the day pointed in the right direction, then begin.

## 6. The Main Daily Workflow

### Morning

1. Open `Today`.
2. Confirm or set the MIT.
3. Confirm the Top 3.
4. Check `Today Load` and `Signals`.
5. Start the first focus block.

### Midday

1. Update tasks honestly.
2. Log habit progress.
3. Move work forward with start dates if it does not belong today.
4. Record friction if something keeps slipping.

### Evening

1. Review what was actually done.
2. Prep tomorrow.
3. Keep only believable work in play.
4. Make a backup if you made major changes.

## 7. What Each Main Page Is For

### Overview

Use `Overview` for orientation, not execution.

Best for:
- checking system health
- seeing reliability
- spotting overdue work
- noticing open habits
- seeing what needs attention before you decide anything

Do not manage details here.
Use it to understand the state of the system.

### Today

`Today` is the center of the app.

Use it for:
- MIT selection
- Top 3 control
- starting focus
- seeing active work
- seeing signals like overdue or neglected items
- logging daily habits
- shrinking the day when overloaded
- prepping tomorrow

If you only use one page most days, use `Today`.

### Inbox

Use `Inbox` for fast capture before deciding what something is.

Then triage each item into:
- Task
- Goal
- Note
- Someday
- Delete

Good rule:
- capture first
- decide later

### Tasks

Use `Tasks` for management, not for running the day minute by minute.

Best for:
- creating and editing tasks
- assigning domains
- setting due dates
- setting start dates
- linking goals
- managing subtasks
- managing recurring task templates
- logging friction

The Tasks page is where you shape the system.
The Today page is where you execute it.

### Habits

Use `Habits` for repeated behaviors that matter.

Best for:
- daily behaviors
- weekly targets
- every-N-days behaviors
- minimum versions
- honest skips
- recovery and consistency

Use habits for repeatable behavior.
Use tasks for one-off commitments.

### Goals

Goals should create action, not just look inspiring.

Every useful goal should have:
- a clear title
- a next action
- a review date
- a health state
- a blocked reason if needed

If a goal has no next action, it is not ready to help you.

### Notes

Use `Notes` for thinking.

Good uses:
- journaling
- project thinking
- reflection
- planning
- lessons learned
- deeper writing that should not become tasks yet

Simple rule:
- Notes = thinking
- Tasks = doing

### Calendar

Use `Calendar` to inspect patterns over time.

Best for:
- overloaded days
- habit consistency
- due-date clustering
- activity patterns

### Weekly Review

Use `Weekly Review` to reset the system once per week.

Best for:
- reviewing what moved
- seeing what slipped
- understanding why it slipped
- choosing next week's focus theme
- generating next-week commitments
- deciding what should be dropped

This page should change your next week.

### Templates / Kits

Use templates for repeated setups that deserve one click.

Good examples:
- morning reset
- shutdown routine
- deep work block
- weekly planning block
- admin hour

### Analytics

Use analytics to learn, not to admire charts.

Best for:
- spotting repeated overload
- noticing reliability changes
- seeing domain imbalance
- learning where focus or consistency breaks

### Settings

Use `Settings` for:
- backups
- restore preview
- recovery bin
- display tuning
- domain profiles
- export
- safety actions

## 8. How To Use Tasks Well

Every task can help or hurt the system.

### Write tasks clearly

Good tasks:
- Send project update email
- Call the mechanic
- Outline lesson plan
- Prepare invoice draft

Bad tasks:
- Fix my life
- Work on project
- Be productive
- Handle everything

### Understand due date vs start date

- `Due date` = when the task should be finished
- `Start date` = when the task should begin appearing in `Today`

Use start dates aggressively.
They are one of the best ways to keep `Today` clean.

### Use MIT and Top 3 properly

- `MIT` = the one task that makes the day real
- `Top 3` = the short list that defines success today

Best practice:
- 1 MIT
- 3 Top 3 tasks max

If your Top 3 is unrealistic, the system becomes noise.

### Use recurring tasks correctly

Recurring tasks are best for scheduled repeat commitments, not habits.

Good examples:
- weekly planning
- pay bills
- monthly check-in
- recurring admin

Use recurring tasks when the work is still a task.
Use habits when the work is really a behavior pattern.

## 9. How To Use Habits Well

The habit system is strongest when you stay honest.

### Good habit examples
- Read 20 minutes
- Stretch 10 minutes
- Train every other day
- Walk 30 minutes
- Journal one paragraph
- Publish 3 times per week

### Use minimum version properly

Example:
- full habit: workout 45 minutes
- minimum version: 10 pushups

The minimum version is for preserving identity and continuity.
It is not for pretending a half-hearted day was a full success.

### Use skip reasons honestly

Good skip reasons:
- low energy
- sick
- travel
- overloaded
- disrupted routine
- forgot

These are not excuses.
They are feedback.

## 10. How To Use Goals Well

A goal should always answer:
- what am I trying to move?
- what is the next action?
- when will I review this again?

Good goal examples:
- Build a consistent training routine
- Ship the first paid product
- Improve sleep regularity
- Strengthen family follow-through

If a goal is stale:
- update the next action
- mark it blocked
- change the health state
- or stop pretending it is active

## 11. How To Plan Today Properly

Use this order inside `Today`:

1. Check the MIT.
2. Check the Top 3.
3. Review overdue work.
4. Review tasks that have reached their start date.
5. Review recurring work due today.
6. Make sure the board still feels believable.
7. Start focus quickly.

Good question:
- "Can I trust this board?"

If the answer is no, reduce scope until it becomes yes.

## 12. What To Do When You Are Overloaded

Do not respond to overload by adding more structure.
Respond by reducing active scope.

When the app signals recovery or overload:

1. Keep one clear MIT.
2. Reduce Top 3 pressure.
3. Move non-essential work out of today.
4. Do minimum habits if needed.
5. Bring one neglected domain back into play.
6. Protect one clean focus block.

A good recovery day might be:
- one real work task
- one minimum habit
- one reset action
- one overdue cleanup action

The goal is traction, not theatrics.

## 13. How To Use Weekly Review Properly

Recommended Weekly Review order:

1. Open `Weekly Review`.
2. Review completed work.
3. Review what slipped.
4. Identify why it slipped.
5. Pick next week's focus theme.
6. Set next week's Top 3.
7. Check goals that need attention.
8. Apply only the habit adjustments you truly want.

Useful weekly questions:
- What actually moved?
- What kept slipping?
- What was too big?
- What was scheduled too early?
- Which domain got neglected?
- What should not be carried into next week?

The review should produce a cleaner next week, not a heavier one.

## 14. Friction Logging

When something slips, log the real reason.

Useful friction reasons:
- unclear
- too big
- low energy
- overloaded
- blocked
- avoidance
- interrupted
- priority shift

Patterns to watch for:
- tasks too vague to start
- too much hard work on the same day
- repeated low-energy failures
- blocked tasks with no next step
- chronic overload instead of real planning

Friction logging turns failure into usable signal.

## 15. Backup, Restore, and Safety

Best setup:

1. Set a backup folder in `Settings`.
2. Use daily auto backup if you want passive protection.
3. Create named snapshots before risky changes.
4. Use restore preview before replacing live data.

Good snapshot names:
- before-cleanup
- before-import
- before-goal-reset
- weekly-stable-state

Important:
- sync backup files, not the live database
- make a safety backup before large structural changes
- use the recovery bin when available instead of panicking

## 16. Common Mistakes To Avoid

- treating the app like a junk drawer
- activating too many tasks at once
- setting everything as MIT-level important
- using habits for one-off work
- never triaging Inbox
- filling Today with future work instead of using start dates
- hiding friction instead of logging it
- skipping backups before major changes

## 17. Minimum Effective Use

If you want the smallest routine that still gives strong value:

### Every day
- open `Today`
- choose MIT
- confirm Top 3
- run one focus session
- log habits honestly
- prep tomorrow

### Every week
- clear Inbox
- run Weekly Review
- update goals
- create a backup

That is enough to get real value from the app.

## 18. Final Mindset

Use Life OS like a feedback system.

Ask:
- What matters now?
- What should start today?
- What can wait?
- What keeps slipping and why?
- What needs recovery instead of pressure?

If you keep the board honest, the app becomes calm, useful, and trustworthy.
